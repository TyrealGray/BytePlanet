Export 3dmax model to UE4

2015.05.10

building model used to be scope of programmer's work, but when they found out this would slow work down and very repeatedly, they decided to write a tool for this kind of work - maya,3dmax.Nowadays, this work is no need to be done by programmers.But if you are an indie game programmer,you are not so lucky.what?2d?That's also annoying too.

Minecraft is a very good example,models are simple and rectangle.

Anyway,use 3dmax to build models.Do not try to use code to build it,It will be a disaster

recently I learned about how to build a model in 3dmax,but I got some problems when I tried to export into UE4.Especially model animation,it caused so many troubles.But thanks to answerhub and Google,I found a way out. Export bones and animation separate,now let's see how it works.

First create you character model ,and build bones for it

=，=that is my character

and remember do the ki of course

when you got this step,we can export our model to UE4 now.Ue4 support fbx,so we export model and bones by fbx first,and name it man.fbx

remember select those ckeckbox

then,wo should export animation now

as the same,this time also select those checkbox(cook animation)

and name it man_walk.fbx

now we got fbx files now

so first export man.fbx like this

then you can see three of the new files in file browser

leave it ,and continue to export the animation

this time exports man_walk.fbx,remember selecte Skeleton, that should tell the UE4 this animation is work with man.fbx

import animations

now remove two of the animation files,just leave man_walk here

double clike man_walk_anim and you would see that interface.You can right click the time axis and remove some thing if you want

for now everything is ready,create you game ~
