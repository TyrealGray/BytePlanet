Implement blueprint Multicast event in C++

Recently when I was coding, found out that Replicated variable cannot make my charactor's material change in both server and client.

so I research, found out I need Multicast!

Multicast 顧名思義，就是會將這個函數代碼複製到所有client玩家機器上執行。也就是說這個機制是不可靠的。

然而什麼時候需要呢？很簡單，就像我之前說的，加入我要自己的人物做一個笑臉的表情，這裡肯定會涉及改變我人物模型上的材質貼圖，如果這個貼圖是 Replicated的變量，那沒有問題，大家都是同步的。但是貼圖這種東西，一個還好，多了之後服務器都要一一同步，負載就太大了。那麼這個時候，顯然貼圖就不能作為 Replicated的變量。這個時候就需要 Multicast了。

Multicast 把切換貼圖的這個函數執行的消息進行通知，所有的client玩家收到這個消息之後，都會在本地執行一次切換貼圖的事件。這樣，我控制的人物模型player1切換貼圖之後，其他玩家收到消息，也將player1切換貼圖，這樣，大家看起來就又都是一樣的了。
